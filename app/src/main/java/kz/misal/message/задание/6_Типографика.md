
# Типографика.

Классификация типографики в Material Design 3 (M3) — это переход от старых HTML-подобных тегов (H1, H2, H3) к **семантическим ролям**.  

Вы выбираете шрифт не по его размеру, а по тому, какую *функцию* он выполняет на экране.

Шкала M3 состоит из 5 базовых категорий, каждая из которых делится на 3 размера (Large, Medium, Small).

---

### 1. Как классифицировать шрифты (5 категорий)

#### 🪧 1. Display (Дисплейные)
Самые крупные шрифты в приложении. Они используются для коротких, очень важных акцентов.
* **Где применять:** Приветственные экраны (Splash screens), огромные цифры (например, баланс счета или крупная скидка), пустые состояния (Empty states).
* **Стиль:** Часто имеют нестандартный или декоративный шрифт, узкие межстрочные интервалы и отрицательный межбуквенный интервал (буквы стоят плотнее).

#### 📰 2. Headline (Заголовки)
Крупные тексты с высокой контрастностью.
* **Где применять:** Главные заголовки разделов или экранов, крупные модальные окна.
* **Стиль:** Хорошо подходят для коротких фраз на 1-2 строки.

#### 🏷️ 3. Title (Названия)
Шрифты среднего размера, которые организуют контент внутри экрана.
* **Где применять:** Названия карточек товаров/отелей, элементы списков, заголовки в верхней панели (TopAppBar), заголовки внутри диалогов.
* **Стиль:** Более удобочитаемые, чем Headline. Обычно имеют среднюю насыщенность (Medium/Bold).

#### 📖 4. Body (Основной текст)
Рабочая лошадка вашего приложения. Предназначен для длительного чтения.
* **Где применять:** Описания, статьи, текст внутри карточек, длинные сообщения.
* **Стиль:** Максимальная читаемость. Обычная насыщенность (Regular), достаточно просторный межстрочный интервал (line-height).

#### 🔘 5. Label (Подписи и Ярлыки)
Мелкий, утилитарный текст, который помогает пользователю ориентироваться и совершать действия.
* **Где применять:** Текст внутри кнопок, подписи к иконкам в нижнем меню (Bottom Navigation), теги, бейджи, всплывающие подсказки (Tooltips).
* **Стиль:** Часто используется жирное (Bold) или среднее (Medium) начертание, чтобы текст оставался читаемым даже при очень маленьком размере. Имеет увеличенный межбуквенный интервал.

---

### 2. Привязка к библиотечным функциям Compose

Когда вы используете стандартные Composable-функции из библиотеки `androidx.compose.material3`, они **автоматически** подтягивают нужные стили из вашего `MaterialTheme.typography`.

Вот шпаргалка, каким элементам соответствуют дефолтные стили:

| Компонент Compose (M3) | Свойство | Дефолтный стиль типографики |
| :--- | :--- | :--- |
| **`Button` / `OutlinedButton`** | Текст внутри кнопки | `labelLarge` |
| **`Text`** (по умолчанию) | Просто текст на экране | `bodyLarge` |
| **`TopAppBar`** (CenterAligned / Small) | Заголовок панели (title) | `titleLarge` |
| **`LargeTopAppBar`** | Раскрытый заголовок | `headlineMedium` |
| **`AlertDialog`** | Заголовок диалога (title) | `headlineSmall` |
| **`AlertDialog`** | Текст диалога (text) | `bodyMedium` |
| **`Card`** | *У карточки нет дефолтного текста, вы задаете его сами* | (Обычно используют `titleLarge` для заголовка и `bodyMedium` для описания) |
| **`NavigationBarItem`** | Текст вкладки нижнего меню | `labelMedium` |
| **`Badge`** (кружочек с цифрой) | Текст внутри бейджа | `labelSmall` |
| **`AssistChip` / `FilterChip`** | Текст чипса/фильтра | `labelLarge` |
| **`OutlinedTextField`** | Вводимый текст | `bodyLarge` |
| **`OutlinedTextField`** | Всплывающая подсказка (Label) | `bodySmall` (в фокусе) / `bodyLarge` (без фокуса) |

---

### 3. Как это выглядит на практике

Знание этих дефолтов позволяет писать очень лаконичный код. Вам не нужно каждый раз указывать размер и жирность шрифта.

Например, верстка стандартного диалога предупреждения будет выглядеть так:

```kotlin
AlertDialog(
    onDismissRequest = { /* ... */ },
    title = { 
        // Compose сам применит к этому тексту стиль headlineSmall
        Text("Удалить аккаунт?") 
    },
    text = { 
        // А к этому тексту применится bodyMedium
        Text("Это действие необратимо. Все ваши данные будут стерты.") 
    },
    confirmButton = {
        Button(onClick = { /* ... */ }) {
            // Кнопка сама покрасит текст в onPrimary и применит стиль labelLarge
            Text("Удалить") 
        }
    }
)
```

Вы просто настраиваете файл `Type.kt` один раз, и вся библиотека Compose автоматически начинает говорить на языке вашего дизайна.


# Плотность

**Стандартная шкала насыщенности (плотности) шрифтов по возрастанию.**

В веб-дизайне, Figma и мобильной разработке (включая Android) каждому названию традиционно соответствует числовое значение от 100 до 900:

- Thin (Hairline) — 100 (Самый тонкий, почти невидимый)
- Extra Light (Ultra Light) — 200
- Light — 300
- Normal (Regular) — 400 (Стандартный базовый текст)
- Medium — 500 (Чуть плотнее обычного, часто используется для подзаголовков)
- Semi Bold (Demi Bold) — 600 (Полужирный)
- Bold — 700 (Стандартный жирный)
- Extra Bold (Ultra Bold) — 800
- Black (Heavy) — 900 (Самый "тяжелый" и массивный шрифт)

# Кастомные шрифты

Подключение кастомного шрифта в Jetpack Compose — это процесс из двух шагов: 
- сначала мы добавляем файлы в проект, 
- а затем «объясняем» Compose, какой файл соответствует какой толщине (Font Weight).

### Шаг 1: Подготовка файлов и папок

Android имеет строгие правила именования ресурсов. Файлы шрифтов не могут содержать заглавных букв, пробелов или дефисов.

1.  Скачайте нужный шрифт (например, Inter или Roboto с Google Fonts) в формате `.ttf` или `.otf`. Вам понадобятся файлы для каждого начертания, которое вы используете (Regular, Medium, Bold и т.д.).
2.  Переименуйте файлы, используя только строчные буквы и нижнее подчеркивание. Например:
3.  В Android Studio перейдите в папку `app/src/main/res/`.
4.  Нажмите правой кнопкой мыши на папку `res` -> **New** -> **Android Resource Directory**. В поле *Resource type* выберите **font** и нажмите OK.
5.  Скопируйте ваши переименованные файлы `.ttf` в созданную папку `res/font`.

---

### Шаг 2: Создание семейства шрифтов (`FontFamily`) в коде

Теперь нам нужно связать эти файлы с кодом Compose. Откройте ваш файл `ui/theme/Type.kt` (или создайте отдельный файл, например, `Font.kt`, если хотите разделить логику).

Здесь мы создаем объект `FontFamily` и перечисляем в нем все наши файлы, указывая их "вес".

```kotlin
//package com.example.app.ui.theme
//
//import androidx.compose.ui.text.font.Font
//import androidx.compose.ui.text.font.FontFamily
//import androidx.compose.ui.text.font.FontWeight
//import com.example.app.R // Обязательно импортируйте R-класс вашего проекта!

// Объявляем наше кастомное семейство шрифтов
val AppFontFamily = FontFamily(
    Font(R.font.inter_regular, FontWeight.Normal),
    Font(R.font.inter_medium, FontWeight.Medium),
    Font(R.font.inter_bold, FontWeight.Bold),
    // Если у вас есть курсив, его добавляют так:
    // Font(R.font.inter_italic, FontWeight.Normal, FontStyle.Italic)
)

// или
val InterFontFamily = FontFamily(
    Font(R.font.inter_thin, FontWeight.Thin),                  // 100
    Font(R.font.inter_light, FontWeight.Light),                // 300
    Font(R.font.inter_regular, FontWeight.Normal),             // 400 (Базовый)

    // Можно также добавить курсив (Italic) для той же плотности!
    Font(R.font.inter_italic, FontWeight.Normal, FontStyle.Italic),

    Font(R.font.inter_medium, FontWeight.Medium),              // 500
    Font(R.font.inter_semibold, FontWeight.SemiBold),          // 600
    Font(R.font.inter_bold, FontWeight.Bold),                  // 700
    Font(R.font.inter_black, FontWeight.Black)                 // 900
)
```


### Шаг 3: Подключение к Material Typography

Теперь осталось только заменить `FontFamily.Default` на наш новый `AppFontFamily` в настройках M3 `Typography`.

```kotlin
//import androidx.compose.material3.Typography
//import androidx.compose.ui.text.TextStyle
//import androidx.compose.ui.unit.sp

// Шаг 1. Наш кастомный шрифт
val AppFontFamily = FontFamily(
    Font(R.font.inter_regular, FontWeight.Normal),
    Font(R.font.inter_medium, FontWeight.Medium),
    Font(R.font.inter_bold, FontWeight.Bold)
)

// Шаг 2. Настраиваем стили M3, используя AppFontFamily
// Переопределяем стандартную типографику Material 3
val AppTypography = Typography(
    headlineMedium = TextStyle(
        fontFamily = AppFontFamily, // <-- Привязываем нашу семью!
        fontWeight = FontWeight.Bold, // Compose сам выберет inter_bold.ttf
        fontSize = 28.sp,
        lineHeight = 36.sp
    ),
    titleLarge = TextStyle(
        fontFamily = AppFontFamily, // <--
        fontWeight = FontWeight.Bold,
        fontSize = 20.sp,
        lineHeight = 28.sp
    ),
    // ... и так далее для всех стилей
)
```

Material 3 содержит 15 стилей текста. Если вы переопределите только 4 из них, то остальные 11 (например, bodyMedium или labelSmall) продолжат использовать стандартный системный шрифт Android (Roboto).
Это приведет к тому, что в вашем приложении шрифты будут "прыгать": заголовки будут шрифтом Inter, а текст в бейджах или диалогах — Roboto.

fontFamily нужно применять глобально ко всем стилям. Чтобы не писать это 15 раз вручную, в Compose используют элегантный хак с функцией .copy(), либо задают базовый стиль.

```kotlin
// Мы хотим выставить значение параметра 
// 1. Выносим настройку "без лишнего воздуха" в отдельную константу
val appPlatformStyle = PlatformTextStyle(
    includeFontPadding = false
)
// Создаем базовую типографику M3
val defaultTypography = Typography()

// Переопределяем ВСЕ стили разом, применяя к ним наш кастомный шрифт
val AppTypography = Typography(
    displayLarge = defaultTypography.displayLarge.copy(fontFamily = AppFontFamily),
    displayMedium = defaultTypography.displayMedium.copy(fontFamily = AppFontFamily),
    // ... и так далее. 
    // А те стили, где нужно изменить еще и размер, прописываем детально:
    titleLarge = defaultTypography.titleLarge.copy(
        fontFamily = AppFontFamily,
        fontWeight = FontWeight.Bold,
        fontSize = 20.sp
    ),
    bodyLarge = defaultTypography.bodyLarge.copy(
        fontFamily = AppFontFamily,
        fontSize = 16.sp
    )
)
```

Теперь, когда вы используете `MaterialTheme.typography.bodySmall` в вашем компоненте `Message`, 
текст автоматически отрисуется шрифтом `Inter Regular`.


Поскольку ваша типографика (`AppTypography`) уже подключена в `MaterialTheme` в файле `Theme.kt`,
**все стандартные компоненты Compose (кнопки, текстовые поля, карточки) 
во всем приложении мгновенно изменят свой шрифт на ваш кастомный.** 
Ни в одном экране вам не придется писать `fontFamily = AppFontFamily` вручную.

В ней используются абсолютно все элементы нашей дизайн-системы: разные уровни типографики, формы, базовые и акцентные цвета.

**В коде верстки нет ни одного жестко заданного цвета (HEX) или размера шрифта**. Компонент полностью опирается на `MaterialTheme`.

### Как M3 делает этот код мощным:

1.  Если вы переключите телефон в темный режим, эта карточка автоматически станет темно-серой, текст станет белым, а красные и зеленые теги примут пастельные оттенки (если вы настроили их в `DarkColorScheme`), чтобы не резать глаза.
2.  Мы используем `MaterialTheme.typography.labelMedium` для бейджей. Если дизайнер изменит размер, вам потребуется поменять одну строчку в файле `Type.kt`.
3.  **Формы (Shapes):** `Surface(shape = MaterialTheme.shapes.small)` гарантирует, что все маленькие элементы в вашем приложении (теги, микро-кнопки) будут иметь одинаковый радиус скругления. Если вы решите сделать дизайн более "круглым" (как в новых версиях Android), вы просто измените `small` форму в `Shape.kt` с 4.dp на 8.dp.

### Как использовать этот компонент на экране:

В вашем главном Activity или фрагменте (где вызывается `setContent`), вы просто оборачиваете этот компонент в вашу тему:

```kotlin
setContent {
    AppTheme { // Ваша тема, которую мы создали
        // Контейнер на весь экран с цветом фона из темы
        Surface(
            modifier = Modifier.fillMaxSize(),
            color = MaterialTheme.colorScheme.background
        ) {
            Column {
                HotelListingCard()
            }
        }
    }
}
```
