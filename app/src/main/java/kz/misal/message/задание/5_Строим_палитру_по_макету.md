Нужно явно показать студентам, что этот код **нельзя** тащить в продакшен, и научить их исправлять ошибки ИИ, приводя ответ к нашей профессиональной двухуровневой системе.

# Практика: 4 способа извлечь палитру из макета Figma

Получив макет (например, прототип Booking.com), разработчик сталкивается с задачей: как перенести цвета из дизайна в архитектуру Material Design 3 (Compose).
Дизайнеры редко отдают готовую `Theme.kt`, поэтому вам придется провести аудит макета.

Рассмотрим 4 рабочих способа от худшего к лучшему.

### Способ 1: Ручной сбор (Антипаттерн)

Самый примитивный путь — кликать по каждому элементу макета и копировать HEX-коды. 
В правой панели Figma вы увидите раздел *Selection colors*, где собраны все цвета выделенного фрейма.

**Почему это плохая идея:**
В реальных макетах вы обнаружите хаос. 
Например, дизайнер мог использовать 73 разных цвета. 
Из них десять — это почти одинаковые оттенки серого (`#E3E3E3`, `#E4E4E4`, `#E6E6E6`).

Если вы начнете создавать переменные под каждый из них (например, `val Casablanca = Color(0xFFF9BA2B)`), 
вы уничтожите архитектуру приложения. Темную тему на таком коде построить невозможно.

**Вывод:** Используйте этот метод только для точечной проверки одного-двух уникальных элементов, 
но никогда не переносите всю эту свалку в проект. 
Ваша задача как инженера — схлопнуть похожие оттенки в единые токены.

---

### Способ 2: Плагин Design Tokens (Для аналитики)

Чтобы быстро оценить масштаб катастрофы в макете, можно выгрузить все цвета разом.
1. Установите плагин **Design Tokens** в Figma.
2. Запустите его (cmd + / -> Design Tokens).
3. Выберите `Export Design Token File`.
4. В настройках оставьте только галочку `Colors` и нажмите Export.

**Плюсы:** Вы получите структурированный JSON-файл со всеми HEX-кодами.
**Минусы:** В Android (Jetpack Compose) нет встроенного механизма для прямой загрузки таких JSON-файлов в `MaterialTheme`. 
Этот способ хорош для аудита, но писать код вам всё равно придется руками.

---

### Способ 3: Плагин Material Theme Builder (Золотой стандарт M3)

Это официальный инструмент от Google, который идеально подходит для генерации базовой темы.

1. Запустите плагин **Material Theme Builder** в Figma.
2. Пипеткой выберите базовые цвета из макета:
    * **Primary:** Главный синий цвет Booking (`#003C94`).
    * **Secondary:** Желтый цвет для рейтингов и акцентов (`#F9BA2B`).
    * **Tertiary / Neutral:** Опционально подстройте оттенки серого и фонов.
3. Нажмите кнопку **Export** (справа вверху) и выберите **Jetpack Compose (Theme.kt)**.

**Что вы получите:** Плагин сгенерирует готовую математическую шкалу тонов (от 0 до 100) 
и правильно распределит их по `lightColorScheme` и `darkColorScheme`.

**Чего не хватает:** Плагин ничего не знает про уникальные маркетинговые цвета (например, зеленый тег скидки или красную точку уведомлений). 
Их придется добавлять вручную через `CompositionLocal`.

---

### Способ 4: Используем ИИ (Осторожно, галлюцинации!)

Вы можете делегировать рутину нейросетям, скормив им JSON из Способа 2 или описав макет текстом.

**Правильный промпт для ИИ:**
> *"Для макета Booking построй тему M3. 
> Выдели базовые примитивные цвета по шкале тонов и распредели их по `lightColorScheme` и `darkColorScheme`. 
> Специфичные цвета (Genius Badge, Notification Red) вынеси в отдельный data class CustomColors для использования через CompositionLocal".*

**Внимание: Критический аудит ответа ИИ**
ИИ часто ошибается. В своих ответах он может выдать вам код, где цвета называются `SapphireDarkContainerNight` или `CandyAppleRedLight`.

**Никогда не копируйте это в проект!** Приведите ответ ИИ к нашему профессиональному стандарту:

**1. Уровень примитивов (Color.kt)** — Только названия красок и их тональность.
```kotlin
// Правильно:
val Blue40 = Color(0xFF006CE4)
val Yellow50 = Color(0xFFF9BA2B)
val Red40 = Color(0xFFD4111E)
```

**2. Уровень схемы (Theme.kt)** — Распределение по ролям.
```kotlin
private val LightColorScheme = lightColorScheme(
    primary = Blue40,
    secondary = Yellow50,
    // ...
)
```

**3. Уровень Custom Colors** — Изоляция уникальных элементов.
```kotlin
data class AppCustomColors(
    val notificationDot: Color,
    val geniusBadge: Color
)
// ... Подключение через LocalCustomColors.current
```

**Итог:** ИИ и плагины — отличные помощники для извлечения HEX-кодов, 
но финальную архитектуру (Примитивы -> M3 Схема -> Custom Colors) всегда собирает инженер. 
Учитесь анализировать макет, а не слепо копировать цифры.

## Алгоритм 

Можно распечатать или держать перед глазами как чек-лист при старте любого нового Android-проекта.

### Этап 1: Визуальный аудит (Не пишем код, смотрим на макет)

Ваша первая задача — проанализировать дизайн глазами инженера, а не просто кликать пипеткой. 
В макете может быть использовано 73 разных цвета, но вам нужно выделить систему.

**Найдите "Трех китов" (Правило 60-30-10):**

60% (Фоны): Найдите базовый цвет фона приложения и цвет карточек (поверхностей). Часто это белый или светло-серый #E3E3E3.

30% (Primary): Найдите главный цвет бренда. Какого цвета самые большие кнопки (например, синий #003C94 у Booking)?

10% (Secondary): Найдите вторичный акцент. Какого цвета бейджи, звездочки рейтинга или переключатели (например, желтый #F9BA2B)?

**Схлопните мусор:** 
Если дизайнер использовал #E3E3E3, #E4E4E4 и #E6E6E6 для разных подложек, примите волевое решение объединить их в один логический токен. Не тащите в проект десятки оттенков серого!

**Выпишите "Изгоев" (Custom Colors):** 
Найдите цвета, которые не вписываются в основную тему. Это цвета статусов (зеленый успех, красная ошибка), цвета брендов партнеров (синий VK, зеленый WhatsApp) или уникальные плашки (например, фон Genius-статуса).

### Этап 2: Генерация тоновой шкалы (Используем инструменты)

Чтобы не высчитывать оттенки для темной и светлой темы вручную, делегируйте математику алгоритмам Material 3.

Откройте плагин **Material Theme Builder** в Figma.

Вставьте ваш главный цвет (Primary) в качестве базового (Source color). Плагин автоматически рассчитает гармоничную шкалу от 0 до 100 для всех ролей.

Нажмите кнопку **Export** и выберите **Jetpack Compose (Theme.kt)**.

> Альтернатива: Вы можете скормить скриншот или HEX-коды нейросети (ChatGPT/Gemini) с жестким промптом: "Сгенерируй тоновую шкалу M3 и распредели по ролям". 
> Но обязательно проверяйте результат на "галлюцинации" (например, ИИ может выдать имена вроде CandyAppleRedLight).

### Этап 3: Кодинг. Уровень 1 — Примитивы (Color.kt)

- Откройте скачанный код и наведите в нем порядок. В файле Color.kt должны лежать только "банки с краской". Никакой логики!

- Очистите файл от мусорных названий вроде Casablanca или DoveGray.

- Переименуйте переменные по стандарту Цвет + Тон (Яркость).

Например: Blue40 (основной синий), Blue80 (светло-синий), Neutral90 (светло-серый), Neutral10 (почти черный).

```Kotlin
// Пример правильного Color.kt
    val Blue40 = Color(0xFF003C94) // Ваш Primary
    val Blue80 = Color(0xFF6F9BFD) // Для темной темы
    val Yellow50 = Color(0xFFF9BA2B) // Ваш Secondary
    val Neutral90 = Color(0xFFE3E3E3) // Фоны
```

### Этап 4: Кодинг. Уровень 2 — Семантика (Theme.kt)

Теперь берем наши "банки с краской" и заливаем их в слоты Material 3. Создаем две схемы: светлую и темную.

Распределите цвета по логике:

- Кнопки -> primary
- Текст на кнопках -> onPrimary
- Фон карточек -> surface или surfaceContainer (для новых версий M3)
- Серые обводки или неактивные элементы -> surfaceVariant.

```Kotlin
// Пример правильной схемы
private val LightColorScheme = lightColorScheme(
    primary = Blue40,
    onPrimary = Color.White,
    secondary = Yellow50,
    background = Neutral90,
    surfaceVariant = Neutral90
// ...
)
```
### Этап 5: Кодинг. Уровень 3 — Специфичные цвета (Custom Colors)

Берем наши цвета-"изгои" из Этапа 1 и безопасно внедряем их в систему через CompositionLocal.

- Создайте data class AppCustomColors и опишите ваши уникальные цвета (обязательно парами: цвет фона + цвет текста на нем, например success и onSuccess).

- Создайте палитры lightCustomColors и darkCustomColors.

- Оберните вашу MaterialTheme в CompositionLocalProvider и передайте туда кастомные цвета.

```Kotlin
Text(
    text = "Genius Level",
    color = MaterialTheme.customColors.onGeniusBadge,
    modifier = Modifier.background(MaterialTheme.customColors.geniusBadge)
)
```
